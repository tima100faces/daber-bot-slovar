## Воркфлоу — КРИТИЧНО (не нарушать)

**Единый источник правды — GitHub, ветка `main`.**

- **Claude работает ЛОКАЛЬНО.** Задача → ветка (`feat/`, `fix/`, `docs/`) → PR → мёрдж в `main`. **Прямых коммитов в `main` нет.** Не мёрджить самостоятельно — ждать «мёрджи» от Тима.
- **Деплой — pull-based**, на сервере OpenClaw, после мёрджа в `main`:
  ```bash
  cd /root/daber-dict && git pull --ff-only origin main && systemctl restart daber-dict
  ```
  Статику nginx отдаёт сразу — рестарт нужен только для backend-изменений.
- **Сервер — это деплой-таргет, а НЕ писатель.** Старый авто-пуш `scripts/daber-dict-push.sh` отключён. Сервер не коммитит/пушит в `main` сам.
- Если `git pull --ff-only` падает (non-fast-forward / локальные правки на сервере) — **остановись, не форсь**, разберись с расхождением.
- **Перед отчётом о готовности — пуш ветки в GitHub + `git log --oneline -3`** (чтобы Тим видел SHA).

## Второй агент — Hermes (DeepSeek)

На сервере OpenClaw работает второй агент — **Hermes** (DeepSeek). Архитектуру он больше не тянет; оставлен на **мелкие фронтенд-правки** (`static/`).

- Hermes **не делает архитектурных изменений** и **не должен** воевать за `main`.
- Если Hermes правит на сервере — он обязан **commit + push ДО** того, как кто-то делает `git pull`.
- Не предполагай, что состояние сервера = `main`. Перед деплоем проверяй чистоту рабочего дерева на сервере.

## Уровни автономности при коммитах

Тим не читает diff'ы кода. Не показывай `git diff` без необходимости.

- **Уровень 1 (коммить автономно):** новые файлы кода по утверждённой спеке, тесты, docstring'и, мелкие багфиксы в своём коде, форматтеры/линтеры.
- **Уровень 2 (короткое summary в 2-3 предложения, потом коммит):** рефакторинг существующих файлов, новые модули.
- **Уровень 3 (показать `git diff`, ждать явного ОК):** изменения схемы БД и миграции, изменения в `docs/`/`README.md`/`CLAUDE.md`, новые зависимости, `.gitignore`/`.env.example`, удаление файлов.
- **Никогда без явного «да»:** `git push --force`, прямые SQL-изменения боевой БД, деплой, работа с секретами/`.env`.
