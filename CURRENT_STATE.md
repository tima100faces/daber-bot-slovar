## 08.06.2026 — формы глагола в поиске/UI, present-глоссы по лицам, верификация

> Источники сверки от преподавателя (Валерия): **Milog** (milog.co.il, толковый иврит-иврит) и **@ivrit_bot** (Telegram, глаголы, рус. перевод+транскрипция+спряжения). Всё ниже **в проде**, кроме доборки корней Milog (в работе).

### Формы глагола: поиск → модалка (раньше уводило на главную)
- Клик по результату-форме (`verb_form`) открывал отдельную страницу `/verb-form/{id}`. Из-за **отсутствия `location /verb-form` в nginx** путь проваливался в SPA-fallback → отдавалась `index.html` → главная. **Фикс:** добавлен `location /verb-form { proxy_pass …:8090 }` (конфиг nginx **не в репозитории**, живёт на сервере; бэкап рядом). Теперь форма открывается **модалкой** (`showVerbFormModal`, точно по id формы), а страница `/verb-form/{id}` оставлена для прямых/расшаренных ссылок.
- **Единый вид модалок.** Шапка формы приведена к виду карточки слова: стек иврит→огласовка→транслит + `modal__info-grid` (часть речи/время/род/число), затем «значение формы», «исходный глагол», перевод, другие формы, примеры, синонимы.

### Present-глоссы — по всем лицам (а не 3-е лицо)
- Иврит. настоящее безличное (אוהב = «я/ты/он любит»). `conjugation_ru.present_*` были в 3-м лице («любит»). Перегенерены в **«люблю / любишь / любит»** (ед.) и **«любим / любите / любят»** (мн.) у **3389** активных глаголов локальной морфологией **pymorphy3** (без платного API). Спрягается существующая (несовершенного вида) глосса, а не перевод-инфинитив → вид сохраняется. **Пассивы (пуаль/hуфаль) не тронуты** (краткое причастие — лицо не применимо). Бэкап: `~/.daber_ivrit/data/conjugation_ru_backup.jsonl`.

### Поиск по русским спрягаемым формам
- Ввод «люблю»/«любишь» матчит русские глоссы `conjugation_ru` (через `jsonb_each_text` + `_parse_conj_slot` — обратный к `_conj_key`) и возвращает **конкретные ивритские формы** (м.р.+ж.р., т.к. настоящее безличное). Для таких запросов формы показываются вместо инфинитив-карточки того же глагола.

### Верификация качества (источники преподавателя)
- **Глаголы — чисто.** Сверка 105 глаголов (все биньяны, ~1940 форм) против @ivrit_bot: **0 реальных расхождений**; ~21% отличий — наш ктив-мале vs дефективное написание бота (не ошибка). (кэш страниц в `~/.daber_ivrit/milog_cache/`, результат — `milog_roots.jsonl`); по завершении — гейт «корень ⊆ слова», сверка, **список на ревью → запись только в NULL**. ## 07.06.2026 — большой ремонт глаголов (фидбэк преподавателя), качество данных, инфра

> Источник большинства правок — экспертный фидбэк Валерии (преподаватель иврита, админ FB-группы). Всё ниже **в проде**.

### Спряжения глаголов
- **Порядок местоимений.** Раньше формы сортировались по строковым колонкам → каша (мн.ч. раньше ед.ч., ж.р. раньше м.р.). Теперь канонический порядок: ед.ч. (я, ты м/ж, он, она) → мн.ч. (мы, вы, они), с подписями **местоимениями** вместо «1-е м ед».
- **⚠️ Архаичные (устаревшие) формы будущего — СКРЫТЫ.** Формы ж.р. мн.ч. будущего и повелительного на ־נה (`תְּדַבֵּרְנָה`) в современном иврите **не употребляются**. Мы их **прячем на уровне показа**: для «вы»/«они» выводится живая форма м.р. мн.ч. (`תדברו`/`ידברו`). **Данные в БД сохранены** (показ-only, откат тривиален). Pealim это подтверждает: «в современном языке используется форма мужского рода». В `aux-forms` Pealim также есть разговорное ударение (формальное `дибартЕм` vs разговорное `дибАртем`) — пока не используем.
- **Йуд / ктив-мале — 45 536 форм.** `verb_forms.form_he` лежал в дефектном написании (ктив хасер: `דברתי`), что преподаватель и заметила. Залито полное современное написание из Pealim (`דיברתי`) у 3 079 глаголов. Бэкап: таблица `verb_forms_form_he_backup` + дамп `backups/verb_forms_pre_ktiv_2026-06-07.sql`.
- **Поиск по обоим написаниям.** form_he теперь полный (`דיברתי`); если обычный поиск не нашёл глагол, повторяем по «скелету» (снятие ו/י) — находятся и старое дефектное, и полное написание.
- **Транслит с ударением.** Хранится с **ударной гласной ЗАГЛАВНОЙ** (`дибАрти`), на экране рендерится через `fixStress()` строчной с акцентом (`дибáрти`). Обновлено 114 015 форм (90%) из Pealim. Бэкап: `verb_forms_translit_backup`.
- **Пассивы (пуаль/hуфаль).** У них нет инфинитива → карточка брала случайную форму (показывала `מדוברות`). Теперь представляются **причастием наст. вр. м.р. ед.ч.** (`מדובר` medubar) — это и заголовок, и транслит.

## Уроки и ошибки сессии 07.06 (чтобы не повторять)
- **Данные без парного отображения = баг.** Залили транслит в заглавных, но забыли применить `fixStress()` на показе — на морде вылезла сырая заглавная посреди слова. Правило: меняешь формат хранения — сразу проверь все точки отображения.
- **Свободные онлайн-конвертеры PDF портят иврит** (RTL+никуд → буквы и корни разъезжаются, `א-ב-ד`→`א-א-ד`). Это не «плохой конвертер», а природа RTL-текста. **PyMuPDF (позиционный) извлекает чисто.** Для текстовых ивритских PDF — только он, OCR не нужен.
- **Не доверять написанию при матчинге омонимов.** Первый вариант стража (`_ktiv_variants`) чуть не удалил `מחשב` («компьютер») как «форму глагола». Решает **смысл перевода (pymorphy3)**, а не похожесть написания.
- **POS-ошибки оказались в основном из IRIS, а не из пайплайна** (гипотеза уточнилась данными: пайплайн ~2 живых, IRIS ~669).
- **Ручные прогоны enrichment дорогие:** два теста сегодня = ~$5.5 (231 вызов, 30 слов). Полагаться на дневной таймер, не дёргать вручную для проверки.

## Архитектура верификации

### Layer 1 — Морфология + БД (бесплатно)
- Мишкали (морфологические шаблоны): CCaC, CiCCuC, miCCaC, hitpael-префиксы
- Окончания: `ה-` (обычно f), `ים-` (обычно m.pl)
- Кросс-чек: поиск однокоренных слов в БД, сравнение POS/рода
- **Дубликат по написанию (ктив мале/хасер):** предупреждение, если кандидат отличается от существующего слова на одну ו/י (показывает найденное слово + перевод)
- **Совпадение с формой глагола:** предупреждение, если headword = спрягаемая форма существующего глагола (от-глагольные сущ. законны → предупреждение, не блок)
- Фразы (`pos_slug: phrase`) — проверки пропускаются

### Layer 2 — Sonnet (батчинг)
- Все подозрительные слова → один API-вызов
- Каждое слово с `context_sentence` и `context_translation`
- Результат: `verdict: ok | fix` с конкретными исправлениями
- Затраты пишутся в `enrichment_costs` с `model='claude-sonnet-4-20250514'`

## API

### Публичные
- `GET /api/search?q=...` — поиск: иврит (точное+префикс по словоформам), кириллица (перевод + лемматизация запроса через pymorphy3: «люблю»→«любить»), латиница (транслитерация)
- `GET /api/word/{word}?id=&type=` — детальная карточка
- `GET /api/letter/{letter}` — слова на букву
- `GET /api/pos/{pos}` — фильтр по части речи
- `GET /api/random?n=` — случайные слова
- `GET /api/stats` — статистика (кэш 1 час)
- `POST /api/feedback` — сообщить об ошибке
- `GET /api/admin/session` — проверка админа (без 401)

### Слово дня и Викторина
- `GET /api/word-of-day` — детерминированное слово дня (хеш даты, freq 200–10000)
- `GET /api/quiz` — случайное слово + 3 дистрактора той же POS

### Факты
- `GET /api/facts/random` — случайный факт
- `GET /api/facts?page=&limit=` — пагинация
- `GET /api/facts/{id}` — детальная

### Админка (API)
- `GET /admin/api/pending?status=pending&id=N` — список/одно pending-слово
- `GET /admin/api/pending/{id}/preview` — превью с enriched-переводами
- `PUT /admin/api/pending/{id}` — редактирование (pos_slug, gender, number, translation_ru, translation_enriched, notes, ...)
- `POST /admin/api/pending/{id}/approve` — одобрить → вставка в words
- `POST /admin/api/pending/{id}/reject` — отклонить
- `GET /admin/api/pending/{id}/check` — проверка дубликатов
- `DELETE /admin/api/word/{id}?type=word|verb` — удаление (каскад)
- `GET /admin/api/costs/summary` — затраты: today_models, month_models, pricing
- `POST /admin/api/enrichment/run` — ручной запуск enrichment
- `GET /admin/api/enrichment/status` — статус (paused, daily_limit, today_inserted)
